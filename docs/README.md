# Docs 目录说明

`docs/` 是 CodexManager 的正式治理文档目录，不再视为临时文件仓。

目标：

- 让结构治理、决策记录、发布说明、一次性排障报告都能进入仓库主线
- 让新协作者不依赖口头交接也能找到正确文档

## 目录分工

### `docs/plan/`

用于保存实施计划、治理清单、阶段性 TODO。

适合提交的内容：

- 结构治理计划
- 里程碑清单
- 分阶段改造任务

### `docs/decision/`

用于保存决策记录和 ADR。

适合提交的内容：

- 技术选型结论
- 架构边界调整记录
- 发布策略或兼容策略的长期决定

### `docs/release/`

用于保存发布说明、回滚方案、发版验收记录。

适合提交的内容：

- 发布流程说明
- 版本发版检查单
- 回滚与验收手册

### `docs/report/`

用于保存扫描、排障、回归、调研类报告。

适合提交的内容：

- 结构扫描报告
- 兼容性调研
- 关键排障记录
- 回归基线说明

## 提交规则

### 应提交到 Git 的文档

- 对未来协作者仍有参考价值
- 会影响后续开发、测试、发布或排障方式
- 可以作为项目长期事实源的一部分

### 不建议提交到 Git 的文档

- 临时草稿
- 个人过程笔记
- 一次性中间产物
- 本地试验记录

## 忽略规则

当前仓库会忽略以下文档：

- `docs/**/*.tmp.md`
- `docs/**/*.local.md`

如果是正式文档，不要使用上述后缀。

## 命名建议

推荐格式：

```text
yyyyMMddHHmmssfff_主题.md
```

示例：

```text
20260309191759589_长期维护结构优化建议.md
```

## 维护约定

- 新增重要治理文档时，优先放到 `docs/`，不要继续塞进 README。
- 版本历史继续写入 `CHANGELOG.md`。
- 架构总览继续维护在 `ARCHITECTURE.md`。
- 协作规范继续维护在 `CONTRIBUTING.md`。
