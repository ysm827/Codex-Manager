# Каталог документации

`docs/` — официальный каталог основной документации CodexManager.

Цели:
- хранить руководства по эксплуатации, заметки по релизам и правила сопровождения прямо в репозитории;
- помогать новым участникам быстро находить нужные документы без устной передачи контекста.

## Снимок проекта

CodexManager — локальный desktop + service-process менеджер пула аккаунтов и gateway-ретрансляции для сценариев Codex.

- Единое управление аккаунтами, usage и platform key.
- Локальный OpenAI-compatible gateway для Codex CLI, Gemini CLI, Claude Code и сторонних инструментов.
- Поддержка маршрутизации аккаунтов, override профилей моделей и relay через aggregate API.

## Последние изменения

- Текущий актуальный релиз: `v0.2.1` (выпущен 2026-04-12).
- Страница управления моделями стала единым редактором каталога: desktop-версия автоматически перезаписывает `~/.codex/models_cache.json` после изменений, а web-развёртывание показывает кнопку экспорта `models_cache.json`.
- Каталог моделей теперь хранит и редактирует расширенные поля вроде `display_name`, `description`, `supported_reasoning_levels`, `visibility` и `supported_in_api`.
- Web-режим теперь должен запускаться через оболочку `codexmanager-web` с `/api/runtime` и `/api/rpc`; одного статического фронтенда уже недостаточно.
- Для free-аккаунтов без `refresh_token` обновление usage больше не вызывает refresh-token 400.

## Обзор возможностей

- Управление пулом аккаунтов: группы, теги, сортировка, заметки, распознавание и фильтрация банов.
- Пакетный импорт/экспорт: мультифайловый импорт, рекурсивный импорт папок в desktop, экспорт по каждому аккаунту.
- Отображение usage: окна 5 часов + 7 дней, single-window аккаунты и дополнительные buckets (Code Review / Spark).
- Platform key: создание, отключение, удаление, привязка модели, reasoning tier и service tier.
- Aggregate API: создание/редактирование/проверка сторонних relay-upstream с поставщиками и приоритетами.
- Plugin center: built-in, private и custom source режимы с задачами/логами и Rhai-интеграцией.
- Локальный service + gateway: настраиваемый bind/listen и единая совместимая точка входа.

## Быстрый старт

1. Запустите desktop-приложение и нажмите **Start Service**.
2. Откройте **Account Management** и завершите авторизацию аккаунта.
3. Если callback не распарсился автоматически, вставьте callback URL для ручного разбора.
4. Обновите usage и проверьте статус аккаунта.

### OpenCowork

- Репозиторий: [AIDotNet/OpenCowork](https://github.com/AIDotNet/OpenCowork)
- Рекомендуемая связка: используйте OpenCowork для локальных файловых операций, multi-agent выполнения, интеграции с рабочими мессенджерами и desktop-автоматизации, а CodexManager оставьте для управления Codex-аккаунтами, учета usage, platform key и локального gateway entry point.
- Лучше всего подходит: командам, которые хотят разделить "рабочее пространство выполнения / офисную интеграцию" и "управление пулом аккаунтов / вход в gateway".
- Коротко: **OpenCowork больше про выполнение в рабочей среде, а CodexManager — про управление и gateway.**

## Правила по документации

### Документацию стоит коммитить, если она
- остаётся полезной для будущих участников,
- влияет на разработку, тестирование, выпуск или диагностику,
- или служит долгосрочным источником истины для проекта.

### Документацию не стоит коммитить, если это
- временный черновик,
- личные рабочие заметки,
- одноразовый промежуточный файл,
- или локальная запись эксперимента.

## Игнорируемые шаблоны
