# 插件中心最小接入说明

这是一版给第三方或内部仓库快速接入用的最小说明。只要按下面的字段和接口返回数据，就能接入 CodexManager 插件中心。

## 1. 你要接什么

- 前端路由：`/plugins/`
- 市场模式：`builtin` / `private` / `custom`
- 数据格式：插件清单 JSON
- 执行方式：Rhai 脚本

## 2. 最小市场返回

返回顶层数组，或返回：

```json
{ "items": [] }
```

每个插件至少要有：

```json
{
  "id": "cleanup-banned-accounts",
  "name": "清理封禁账号",
  "version": "1.0.0",
  "scriptBody": "fn run(context) { ... }",
  "permissions": ["accounts:cleanup"],
  "tasks": [
    {
      "id": "run",
      "name": "定时自动清理",
      "entrypoint": "run",
      "scheduleKind": "interval",
      "intervalSeconds": 60,
      "enabled": true
    }
  ]
}
```

## 3. 最小字段

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `id` | 是 | 唯一 ID |
| `name` | 否 | 名称，缺省回退到 `id` |
| `version` | 否 | 版本号，缺省 `0.0.0` |
| `scriptBody` | 选其一 | 直接内嵌脚本 |
| `scriptUrl` | 选其一 | 脚本地址 |
| `permissions` | 否 | 权限列表 |
| `tasks` | 否 | 任务定义 |

建议额外补：

- `manifestVersion`
- `category`
- `runtimeKind`
- `tags`

## 4. 最小接口

### Tauri

- `service_plugin_catalog_list`
- `service_plugin_install`
- `service_plugin_update`
- `service_plugin_uninstall`
- `service_plugin_list`
- `service_plugin_enable`
- `service_plugin_disable`
- `service_plugin_tasks_update`
- `service_plugin_tasks_list`
- `service_plugin_tasks_run`
- `service_plugin_logs_list`

### RPC

- `plugin/catalog/list`
- `plugin/install`
- `plugin/update`
- `plugin/uninstall`
- `plugin/list`
- `plugin/enable`
- `plugin/disable`
- `plugin/tasks/update`
- `plugin/tasks/list`
- `plugin/tasks/run`
- `plugin/logs/list`

## 5. Rhai 最小可用函数

如果插件申请了这些权限，就能用对应函数：

- `settings:read` -> `get_setting(key)`, `list_settings()`
- `network` -> `http_get(url)`, `http_post(url, body)`
- `accounts:cleanup` -> `cleanup_banned_accounts()`, `cleanup_unavailable_free_accounts()`

公共函数：

- `log(message)`

## 6. 入口结论

如果你是：

- 官方精选市场，直接返回内置同结构 JSON
- 企业私有市场，返回私有仓库 JSON
- 自定义仓库，保持 `scriptBody` 或 `scriptUrl` 即可

当前最稳的接法是“清单先行，脚本后置”，不要把重型逻辑直接堆进 Rhai。
