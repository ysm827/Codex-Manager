# 账号导入格式与数据库扩展结论

## 结论

- 当前数据库层仍然是 `SQLite-only`，暂不扩展 `PostgreSQL` / `MySQL`。
- 当前账号导入支持“按文件导入”和“桌面端按文件夹导入”两条路径。
- 导入内容本质上只支持 `JSON`，不支持 `CSV`、`Excel`、`YAML`、`XML` 这类非 JSON 格式。

## 数据库扩展结论

- 当前项目的存储实现直接依赖 `rusqlite`，不是只改连接串就能切库的结构。
- `Storage` 直接持有 SQLite 连接，迁移脚本和查询也存在 SQLite 专有语法与行为。
- 因此本轮不推进多数据库适配，继续按 SQLite 维护。

## 当前导入支持什么格式

### 1. 文件类型

- 文件选择导入：支持选择 `.json` 和 `.txt` 文件。
- 文件夹导入：仅桌面端支持，递归扫描目录下所有 `.json` 文件。
- 浏览器模式：不支持文件夹导入。

说明：

- `.txt` 只是文件选择器允许选择，内容仍然必须是合法 JSON。
- 文件夹导入不会扫描 `.txt`，只会读取 `.json`。

### 2. JSON 内容结构

后端实际支持以下三类内容：

1. 单个 JSON 对象
2. JSON 数组
3. 连续 JSON 对象流

“连续 JSON 对象流”指的是文本里顺序放多个完整 JSON 对象，只要整体能被 JSON 反序列化流逐个读出即可。

### 3. 字段兼容规则

每个导入项至少需要以下 3 个令牌字段：

- `access_token` 或 `accessToken`
- `id_token` 或 `idToken`
- `refresh_token` 或 `refreshToken`

可选账号提示字段：

- `account_id` 或 `accountId`
- `chatgpt_account_id` 或 `chatgptAccountId`

字段既可以放在对象顶层，也可以放在 `tokens` 子对象里。

### 4. 支持的对象形态

#### 顶层字段

```json
{
  "access_token": "xxx",
  "id_token": "xxx",
  "refresh_token": "xxx"
}
```

#### `tokens` 包裹字段

```json
{
  "tokens": {
    "access_token": "xxx",
    "id_token": "xxx",
    "refresh_token": "xxx",
    "chatgpt_account_id": "acc_xxx"
  }
}
```

#### 数组批量导入

```json
[
  {
    "accessToken": "xxx",
    "idToken": "xxx",
    "refreshToken": "xxx"
  },
  {
    "tokens": {
      "access_token": "yyy",
      "id_token": "yyy",
      "refresh_token": "yyy"
    }
  }
]
```

## 当前不支持什么格式

- `CSV`
- `XLS` / `XLSX`
- `YAML` / `YML`
- `XML`
- 非 JSON 纯文本
- 浏览器模式下按文件夹导入

## 导入行为补充

- 导入不是“只新增”，而是“新增或更新”。
- 系统会尝试从 `id_token` / `access_token` 中提取 `chatgpt_account_id`、`workspace_id`、邮箱等信息。
- 桌面端按文件夹导入时，会递归扫描子目录中的 `.json` 文件。

## 代码依据

- 文件夹导入命令：`apps/src-tauri/src/lib.rs`
- 文件夹递归扫描：`apps/src-tauri/src/app_storage.rs`
- 后端导入解析：`crates/service/src/account/account_import.rs`
