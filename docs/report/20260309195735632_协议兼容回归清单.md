# 协议兼容回归清单

## 固定覆盖范围

每次涉及协议适配、转发、流式转换、tools 聚合时，至少回归以下维度：

1. `/v1/responses`
2. `/v1/chat/completions`
3. stream
4. non-stream
5. tools / `tool_calls`

## 最小矩阵

| 入口 | 模式 | 工具 | 期望 |
|---|---|---|---|
| `/v1/chat/completions` | non-stream | 无工具 | 返回 OpenAI Chat 兼容 JSON |
| `/v1/chat/completions` | non-stream | 有工具 | 保留 `tool_calls`，不能只剩 `message.content` |
| `/v1/chat/completions` | stream | 无工具 | SSE 增量正常结束 |
| `/v1/chat/completions` | stream | 有工具 | 工具增量、聚合结果与结束事件正确 |
| `/v1/responses` | non-stream | 无工具 | 返回 Responses 兼容 JSON |
| `/v1/responses` | non-stream | 有工具 | `output` / 工具结果结构不丢失 |
| `/v1/responses` | stream | 无工具 | SSE/事件流正常结束 |
| `/v1/responses` | stream | 有工具 | 工具名缩短与响应还原正确 |

## 人工核对点

- 请求日志中的原始路径与适配路径是否正确
- `tool_calls` 是否在最终聚合 JSON 中保留
- Responses 转 Chat/Chat 转 Responses 时是否出现字段丢失
- 工具名缩短后是否还能正确还原
- Cherry Studio、OpenClaw、Claude Code 这类兼容客户端是否仍可正常工作
